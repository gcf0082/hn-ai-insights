### 1. [Gemini API 密钥被盗刷 8.2 万美元](https://news.ycombinator.com/item?id=47231469)

**英文标题:** Stolen Gemini API key racks up $82,000 in 48 hours

**分数:** 84 | **评论:** 50 | **来源:** [LLM Horrors](https://llmhorrors.com/all/gemini-stolen-api-key-82k/)

#### 核心内容
- 开发者 Andras Bacsai 的 Google Cloud API 密钥被盗，48 小时内产生 82,314 美元 Gemini 费用
- 正常月度支出仅约 180 美元，异常消费增长超过 450 倍
- 事件根源：API 密钥未设置消费上限和消费警报

#### 关键讨论
- 社区强烈建议所有云服务 API 密钥必须设置 billing cap（消费上限）
- 开发者呼吁云服务商应默认启用消费限制而非可选功能
- 有人指出这是"serverless horrors"典型案例，无限制计费是重大风险

#### 分析价值
⭐⭐⭐⭐⭐ 为所有使用云 API 的开发者敲响警钟，消费上限应成为 API 密钥管理的标准配置，否则可能一夜破产
